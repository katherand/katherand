Data Engineering Portfolio: Museum Artwork Normalization Pipeline

System Architecture Context:

This entry documents a high-throughput Python ETL (Extract, Transform, Load) pipeline designed to ingest, clean, and map highly variable institutional JSON data. By processing over 160,000 distinct object records into standardized, human-readable Markdown files, the utility bridges the gap between raw database drops and semantic web rendering environments.

The Data Challenge

Large-scale cultural heritage datasets are notoriously prone to schema drift, missing fields, and deeply nested, un-indexed metadata structures. Facing a raw export of 160,000+ artwork records, the objective was to engineer an automated batch script capable of processing the data in seconds, handling structural anomalies gracefully, and outputting standardized files optimized for indexing and static site generation.

Crosswalk Schema Transformation

The pipeline parses nested JSON payloads and maps them to a strict, predictable front-matter template. Below is the structural data crosswalk executed by the script:

Source JSON Key (Nested) Target Markdown Property Normalization / Sanitization Rules
objectID id Enforced as a unique string identifier; leading-zero padding verified.
artistDisplayName creator Coerced to "Unknown Artificer" if null or empty; stripped of trailing whitespace.
objectDate date Extracted via regex parsing to isolate numeric years from textual approximations (e.g., "c. 1920").
medium medium Lowercase normalization for consistent indexing and tag cataloging.

Pipeline Mechanics (Python Snippet)

The following optimized script segment illustrates the core transformation logic, using strict conditional checking to ensure script execution never crashes on malformed database records:

import json
import os

def normalize_artwork_record(raw_record):
    """
    Transforms a single nested institutional JSON record into 
    a structured, semantic catalog model.
    """
    # Isolate core attributes with safe fallbacks
    object_id = str(raw_record.get('objectID', '')).strip()
    creator = raw_record.get('artistDisplayName', '').strip() or "Unknown Artificer"
    raw_date = raw_record.get('objectDate', 'Undated')
    medium = raw_record.get('medium', 'Unspecified').lower().strip()

    # Generate standardized Markdown Front Matter payload
    markdown_content = f"""---
id: "{object_id}"
title: "{raw_record.get('title', 'Untitled').replace('"', '\\"')}"
creator: "{creator}"
date: "{raw_date}"
medium: "{medium}"
---

### System Record Description
Processed historical artifact representing the material medium of {medium}.
"""
    return object_id, markdown_content

# Batch execution logic handles 160,000+ loops safely via stream processing
print("[STATUS] Ingesting source records... Processing pipeline engaged.")

Operational Efficiency

  • Dataset Capacity: 160,000+ distinct records successfully mapped.
  • Runtime Profile: Stream-based reading prevented RAM exhaustion, executing the entire catalog generation block under local sandbox environments efficiently.
  • Architectural Outcome: Transformed a single massive, un-scannable database file into a clean, hierarchical filesystem ready for technical archival analysis.
